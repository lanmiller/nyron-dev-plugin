# Тест-дизайн: блок «Как тестировать» и выбор техники

Блок «Как тестировать» в тикете — опциональный, но ОБЯЗАТЕЛЕН для багов и
фич с пользовательским поведением. Он отвечает не «что готово» (это DoD), а
«какими случаями это проверяем». Пишется на груминге или волной перед TDD —
из него рождаются имена тестов.

## Правило выбора техники (по форме задачи)

| Форма задачи | Техника | Что даёт |
|---|---|---|
| Входы с диапазонами/вариантами (размеры, лимиты, форматы, роли) | **Equivalence Partitioning + граничные значения** | классы эквивалентности: по одному кейсу на класс + границы (0, max, max+1, пусто, кириллица/эмодзи…) |
| Есть состояния и переходы (статусы, режимы, жизненный цикл) | **State Transition** | таблица/диаграмма «состояние × событие → новое состояние»; кейсы = все валидные переходы + запрещённые (должны отбиваться) |
| Поведение зависит от КОМБИНАЦИЙ условий (флаги, роли, тарифы) | **Decision Table** | таблица «условия → действия»; кейсы = каждый столбец таблицы |
| Просто сценарий пользователя | сценарные шаги | happy-path + 1-2 отклонения |

Комбинировать можно: у фичи со статусами и ролями будет STD + Decision Table.

## State Transition и карта флоу

Посекундная карта живого прогона (методика live-отладки,
`ai-evolve-docs-test/n8n/playbook-flow-for-methodologist.md`) — это фактическая
ТРАССА переходов системы: «время → событие → слой → этап». Из неё STD строится
механически: строки трассы группируются в состояния, события — в переходы.
Поэтому:
- для фич с жизненным циклом (карточка плейбука: collect → catalog → building →
  ready → playing; тикет: Бэклог → В работе → …) в блок «Как тестировать»
  класть мини-STD таблицей;
- расхождение живой трассы с STD = баг либо дыра диаграммы — оба исхода ценны.

## Формат блока в тикете

**Как тестировать** (техника: <EP / STD / DT / сценарий>):
- <кейс 1 — из класса/перехода/столбца>
- <кейс 2>
- негативные: <запрещённый переход / невалидный класс — что должно произойти>

Пример (STD, кнопка «Перезапустить»):
| Состояние карточки | Событие | Ожидание |
|---|---|---|
| первая сборка, до прохождения | вход в прохождение (авто) | признак снят, кнопка появится после прохождения |
| прошли плейбук | открыть карточку | кнопка «Перезапустить» видна |
| прогон идёт (live) | смотреть карточку | кнопки нет (защита от рестарта посреди прогона) |
| негатив: признак залип | — | воспроизводит баг; после фикса невозможно |
