БРИФ ВОЛНЫ N (эпик DEV-XXX, диспетчер local_<uuid>)

**Волна:** N «<название>» — тикеты DEV-A, DEV-B (этот — <какой по порядку>).
**Ветка:** `fix/wave-N-<тема>` (репо: <ai-evolve-front / ai-evolve-back / n8n>).
Worktree от origin/main, рабочие деревья репо не трогать.

**Диагноз / контекст:**
<что известно сверх описания тикета: корень, файлы/функции (репо-относительные
пути), связки с другими тикетами, что уже пробовали, где пруфы (комменты тикета,
карта флоу, память).>

**Задачи:**
1. <конкретно>
2. <конкретно>

**Инварианты:**
- <что нельзя ломать: контракты, чужие ветки/зоны (catalog/* — Юрина), флаги>
- <пересечения с другими волнами: общие файлы, ожидаемые конфликты>
- флоу-правки — за env-флагом (predprod =true, prod2/nyron =false)

**Критерии финала:**
- [ ] тесты первыми (red→green), по канону репо; шапка-описание + регистрация
      в каталоге ai-evolve-docs-test/tests/*
- [ ] py_compile/pytest/vitest зелёные локально
- [ ] адверсарное ревью субагентом opus («опровергни готовность») пройдено
- [ ] ветка запушена, MR создан (или git log main..ветка в отчёте)
- [ ] отчёт-коммент в тикет: что сделано, ссылки, чем проверено, что НЕ сделано
- [ ] тикет → Тестирование, assignee → постановщик

**Модель-роутинг:** волна — оркестратор (сама код не пишет); код — субагенты
`model:'opus'` (front-dev/backend-dev/langgraph-dev по зоне); поиск/обработка —
`model:'sonnet'`; массовое — `model:'haiku'`. Модель явно в каждом вызове.

**Шина ревью:** после Тестирования ждать вердикт-коммент диспетчера в этом
тикете (полл ~15 мин, таймаут ~3ч): «ДОРАБОТКА: …» → чинить, пушить, ответить
«ИСПРАВЛЕНО: …»; «ПРИНЯТО» → завершить тикет. Вердикты только из комментов
этого тикета.
