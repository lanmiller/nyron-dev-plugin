# Changelog — nyron-dev

Формат: одна запись на версию; что изменилось и зачем (одной-двумя строками
на пункт). Бамп версии в plugin.json = запись здесь, тем же коммитом.

## 0.2.16 — 2026-07-19

- **Фильтр эха в будка-вотчерах** (live-находка волны: вотчер будился о
  собственный hub_post): строки from=<я> игнорировать, после своего поста —
  перезамер базы и перевзвод. Зеркало давнего правила эха Jira-комментов,
  теперь на обоих будка-вотчерах (диспетчер и волна).

## 0.2.15 — 2026-07-19

- **Будка симметрична** (live-кейс: вердикт лежал в будке, а волна поллила MR
  15-минутками): вердикт-ожидание волны — теперь БУДКА-ВОТЧЕР основным каналом
  (until-loop ~15с; латентность секунды), Jira-вотчер — дубль, sleep-полл —
  только last resort. Диспетчер ОБЯЗАН дублировать вердикт hub_post'ом
  («ВЕРДИКТ DEV-X: ...»). Пуш-обмен в обе стороны, поллинги умерли.

## 0.2.14 — 2026-07-19

Пакет по фидбеку live-юзания (3 дыры координации + маячок):

- **hub_post-чекпойнты обязательны** для волн: «взял DEV-X» / «сдал DEV-X» /
  «волна сдана» / «блокер» — пульс для вотчера; отчёты остаются ТОЛЬКО в Jira.
- **Будка-вотчер диспетчера** — в штатный набор вотчеров (until-loop на рост
  .nyron-hub/messages.jsonl, пробуждение на чекпойнты за секунды).
- **Sol обязателен на трёх уровнях**: батч-чек ПОСТАНОВОК тикетов волны
  (plan-challenge.sh по файлу с JTBD/DoD) + план/нарезка (все волны; мелким —
  одним слитым вызовом, но не скипать) + cross-review внутри волны (как было).
- **hub_status — маячок первым действием волны**: видно «плагин подцепился»
  и чужие брони; тулз нет → не блокер, зафиксировать в план-комменте.

## 0.2.13 — 2026-07-19

- **Codex-вызовы — фоном** (фидбек live-юзания: синхронный cross-review на

## 0.2.9 — 2026-07-19

- **Смежные узлы в ревью и план-челлендже**: Sol обязан проверять, переживает
  ли изменение/план соседние РЕАЛЬНЫЕ режимы, трогающие те же данные (типовой
  баг «спека про создание — рестарт не отработан»); только с пруфом из кода,
  гипотетика запрещена. Карта режимов — новая секция `flows:` конфига проекта
  (у ai-evolve заполнена: play/restart/edit/forward/publish→use/версии/reset/

- **DRY/KISS/SOLID в cross-review**: DRY-дубль легаси = блокер (канон
  «дорабатывай легаси»); KISS/SOLID/стиль — блокер только если прячет баг,

## 0.2.8 — 2026-07-19

- **План-челлендж второй моделью** (scripts/plan-challenge.sh + шаг груминга
  в wave-protocol): для крупных задач на уровне спеки/раскадровки — Fable
  штурмует узкие места, Sol сверяет план с реальным кодом (read-only): ложные
  предпосылки, дубли легаси, пересечения нарезки, дыры DoD, более простой
  путь. Вердикт «ПЛАН: ОК|РИСКИ» правит нарезку ДО чипов. Мелочи — не нужен.
  Пайп целиком: план (Fable+Sol) → TDD → код (Opus) → адверсарка → ревью Sol
  → [qa-run — 0.3.0].

## 0.2.7 — 2026-07-19

  запрещён» — без изменений, работает при любом мейне.

## 0.2.0 — 2026-07-18

Двухшаговая система «продукт → доставка» (по записке nyron-vyvody): плагин
превращается из набора delivery-скиллов в переиспользуемую основу для любого
проекта. 4 новых скилла, MCP-сервер, конфиг-слой.

- **`cross-review` (новый скилл + scripts/cross-review.sh)** — кросс-ревью
  другой моделью: GPT через codex CLI (ChatGPT-подписка, read-only песочница)
  проверяет код Клода; «кто писал — тот не проверяет». Вердикт машиночитаем
  (`ВЕРДИКТ: ПРИНЯТО|ДОРАБОТКА`), встроен в конвейер волны обязательным шагом
  перед сдачей в Ревью. Разблокирует «доезжает до готово без глаз человека».
- **`intake-groom` (новый скилл)** — чистильщик входящих: сырая фича →
  draft-тикет по jira-task-standard силами sonnet/haiku + открытые вопросы
  автору; сырьё сохраняется дословно комментом; draft в волны не входит.
- **`team-roles` (новый скилл)** — ролевые карточки: продакт, UX-дизайнер,
  дата-аналитик, техлид, SRE-лид, QA (миссия/вход/выход/чек-лист/границы).
  Generic в плагине, проектное (модель, «куда смотреть», заметки) — из
  конфига `roles.*`. Диспетчер/ревьюер/чистильщик ролями не являются.
- **`project-config` (новый скилл + шаблон)** — конфиг проекта
  `.claude/nyron-dev.md` (+`.local.md` оверрайды): тип development/research,
  хранилище задач jira/files, карта моделей по ролям, движок ревью, роли.
  «Как делать — в скилл, чем проекты отличаются — в конфиг»; один плагин на
  Evolve/Betzo/psylua.
- **`nyron-hub` (новый MCP-сервер, hub/server.mjs)** — «будка»: zero-deps
  Node stdio-сервер в плагине; шина сообщений агентов (hub_post/read), бронь
  файлов с детекцией пересечений (hub_lock/unlock), очередь мержа по репо —
  вливаем по одной от головы (hub_merge_join/leave). Состояние — файлы
  `.nyron-hub/` проекта; Jira будка НЕ трогает (Jira = долгоживущее,
  будка = быстрое служебное). Jira остаётся per-project MCP.
- **nyron-waves интегрирован с новым**: конфиг первым ходом; кросс-ревью
  обязателен в финале тикета; брони файлов при старте правок; мерж пачки —
  по одному через очередь будки с ребейзом на свежую основу; `hub_status`
  на каждом пробуждении диспетчера. Шаблон wave-rules обновлён (канон в
  docs-test синхронизировать при первом использовании).
- README: два борта, жизненный цикл от сырья, требования Node/codex,
  подключение к новому проекту.

## 0.1.3 — 2026-07-17

Итоги обкатки ночной пачки волн 17.07 (6 волн, 17/17 тикетов приняты к утру).

- **nyron-waves: активный мониторинг Jira диспетчером** (закрыт ночной пробел —
  волны перевели 17 тикетов «В работе» за 5 минут, диспетчер узнал от
  пользователя): контрольный JQL-чек через 5–10 мин после старта чипов; JQL по
  тикетам волн ПЕРВЫМ действием на каждом пробуждении; опциональный Jira-вотчер
  на REST-токене (дефолт остаётся бес-токенным).
- **nyron-waves: гибрид вотчеров — канон вместо sleep-полла** (латентность
  сдача→ревью ~3 мин вместо 15–25): фоновые until-loop'ы будят сессию по
  событию, ScheduleWakeup — только страховочный пульс. Правила: фильтр эха
  своих комментов, перевзвод только отдельным трекаемым run_in_background,
  волна в ожидании вердикта по А пилит Б.
- **nyron-waves: коммит-конвенция волн** — заголовок коммита начинается с ключа
  тикета (`DEV-XXX: …`), пачка тикетов — все ключи списком; те же ключи в MR
  и отчёт-комменте.
- **Atlassian-доступ канонизирован**: только плагинный MCP (`plugin:atlassian`,
  грант на nyron), проверка гранта первым ходом — в профиле бывает второй
  коннектор без гранта, ночью сессии в него втыкались.
- **nyron-waves: уроки эксплуатации** — зависшие ревью-агенты (перезапуск узким
  скоупом, после 2 фейлов — руками), критерий смерти волны и takeover поверх
  её веток, критичные правила в бриф ДО старта, vitest-грабля worktree фронта,
  бранч-гигиена после мержа пачки; тезис «Jira = событийная шина» вписан якорем.
- **live-epic-testing: грабли мониторов** — буферизация в пайпах
  (`--line-buffered`/`stdbuf`), отправка чата только Enter-keydown, второй
  коннект юзера крадёт HITL.
- README: известная грабля рассинхрона кэша Desktop ↔ CLI; nyron-dev/README —
  актуальная установка (отдельный репо вместо docs-репо) и требования доступов.
- Канон `ai-evolve-docs-test/shared/wave-rules.md` синхронизирован с шаблоном
  (коммит-конвенция, вотчер вместо полла, vitest-грабля, проверка Atlassian).
